# Animetro Website Content Pipeline

This website is a static HTML site. It does not read Google Sheets at runtime.

The approved content source used by the static build is:

`content/website-content.csv`

When these environment variables are available, the sync script first exports
the latest Google Sheet content into `content/website-content.csv`:

```text
GOOGLE_SHEET_ID
GOOGLE_SERVICE_ACCOUNT_JSON
```

To verify read-only access to a shared Google Drive folder, also set:

```text
GOOGLE_DRIVE_FOLDER_ID
```

Optional:

```text
GOOGLE_SHEET_TAB_NAME
```

Default tab name:

```text
Website Copy
```

To sync approved spreadsheet content into the website files, run:

```bash
python3 scripts/sync_website_content.py
```

The sync script updates:

- shared header logo taglines across English and Chinese pages
- English and Chinese home hero content
- English and Chinese home Core Services card titles
- English and Chinese Process section
- English and Chinese Testimonials section
- English and Chinese home contact CTA
- English and Chinese About / Team section
- English and Chinese Contact details
- `dist/` deployment output

Current Resources, Insights, Events, and Services pages are static HTML pages.
Their latest approved content is present in `en/`, `zh/`, and `dist/`, but they
are not regenerated by the sync script yet.

For deployment, upload the contents of the clean deploy folder or `dist/` after
running the sync script.

## Automatic GitHub Sync

The workflow `.github/workflows/sync-website-content.yml` can be run manually
from GitHub Actions and also runs daily. It:

1. exports the Google Sheet to `content/website-content.csv`
2. runs `scripts/sync_website_content.py`
3. commits changed source website files automatically

Required GitHub repository secrets:

```text
GOOGLE_SHEET_ID
GOOGLE_SERVICE_ACCOUNT_JSON
GOOGLE_DRIVE_FOLDER_ID
```

Optional GitHub repository variable:

```text
GOOGLE_SHEET_TAB_NAME
```

## Google Drive Folder Setup

The Drive folder is read-only. The sync script only reads metadata from files
inside the configured folder and does not download, replace, or change website
images.

To find the folder ID:

1. Open the folder in Google Drive.
2. Copy the browser URL.
3. Use the value after `/folders/`.

Example:

```text
https://drive.google.com/drive/folders/1_-jbZCPZBW8HgVVGdnOr6WsOw95NJhS_
```

Share the folder with this service account as a Viewer:

```text
animetro-sheet-logger@hardy-abode-499712-e7.iam.gserviceaccount.com
```

Add this GitHub repository secret:

```text
GOOGLE_DRIVE_FOLDER_ID = 1_-jbZCPZBW8HgVVGdnOr6WsOw95NJhS_
```

Add the same value in Vercel:

```text
Settings > Environment Variables > GOOGLE_DRIVE_FOLDER_ID
```
